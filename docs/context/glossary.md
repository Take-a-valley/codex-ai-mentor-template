# Glossary

このファイルは、プロジェクト内で使う用語を整理します。

## 用語一覧

| 用語 | 意味 | 備考 |
|---|---|---|
| プロジェクト | このテンプレートを使って作成する学習対象の開発プロジェクト | 題材に応じて意味を具体化する |
| MVP | Minimum Viable Product | 最初に作る最小構成 |
| 要件定義 | 何を作るか、誰のために作るかを整理する工程 | 実装前に行う |
| 基本設計 | システム全体の構成や画面・機能の概要を整理する工程 | 外部設計とも呼ばれる |
| 詳細設計 | 実装に近い粒度で処理やデータ構造を整理する工程 | 内部設計とも呼ばれる |
| レビュー | 作成物の問題点や改善点を確認すること | 設計書・コードの両方に対して行う |
| 永続コンテキスト | AIが次回以降も参照するための文脈情報 | `docs/context/` や `docs/logs/` に保存 |
| クライアントモード | ユーザーがクライアント役、AIがPM役として要件を整理するモード | `【クライアント】` で開始 |
| 終了報告 | 作業終了時にログと次回タスクを整理する運用 | `終了報告` で実行 |
